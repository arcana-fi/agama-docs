**NAV Oracle — gnark ZK proof (Go)**

The only software Agama builds. Everything else is Lagoon.

| Component | File | Description |
|-----------|------|-------------|
| Connector | enygma_client.go | Reads invoice data from Privacy Node via Rayls API + view key |
| Circuit | nav_circuit.go | gnark Groth16 circuit. Proves NAV = Σ [purchase_price + spread × accrual_factor]. Tranching: senior (≤30d) / mezzanine (30-60d) / junior (>60d) |
| Setup | compile.go | Compiles circuit. Generates AgamaVerifier.sol. Run once |
| Publisher | onchain_relay.go | Publishes (NAV, TrancheNAV, π) on Public Chain. Gas in USDr |
| Config | nimofast.config.json | View key, Privacy Node RPC, update interval (daily V1) |

**AgamaOracle.sol** on Rayls Public Chain

Wrapper around gnark-generated AgamaVerifier.sol. `updateNAV(TrancheNAV, proof)` verifies ZK on-chain and updates share price. No multisig — the proof replaces the signature. Anyone can call `verifyProof()` to check.

**Lagoon curator setup**

agaINV vault ERC-7540 on Rayls Public Chain. KYC whitelist managed manually in V1. Lagoon handles deposit/withdrawal async, fee computation, and reporting.

**Frontend**

agama.finance — branded institutional frontend. Lagoon app: automatic listing upon deployment.
